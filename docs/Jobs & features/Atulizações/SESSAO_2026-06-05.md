## 1. Truncamento de Fontes

### Objetivo

Reduzir o contexto enviado ao modelo de síntese, priorizando as fontes mais relevantes e evitando sobrecarga de token.

### Implementação

**Arquivo:** `js/writer.js`

**Função criada:** `truncateSources(sources, maxSources = 5)`

**Comportamento:**
- Se `sources.length <= 5`: retorna todas
- Se `sources.length > 5`: ordena por relevância/priority/quality.score e pega top 5
- Log: `[WRITER] sources truncated: X→5`

**Integração em chat-models.js:**

Linha ~2117, antes de construir o material de síntese:

```javascript
const truncatedPages = typeof truncateSources === 'function'
  ? truncateSources(fetchedPages, 5)
  : fetchedPages.slice(0, 5);
```

**Resultado:**
- `runDeterministicResearchRoute` agora sempre entrega no máximo 5 fontes para a síntese
- Redução de contexto: ~40-60% em queries com muitas fontes

## 2. Sistema generateAndRefine

### Objetivo

Criar um **rascunho interno** (não visível ao usuário) via Haiku, depois **refinar** esse rascunho com Writer, aplicando:
- Tom Worion (sem frases de atendimento)
- Verificação de afirmações vs evidência real (toolResults)
- Filtro de refutações do usuário
- Remoção de frases proibidas

### Fluxo

**PASSO 1 — RASCUNHO:**
```
Modelo: Claude Haiku 4.5
Max tokens: 300
Prompt: "Dado contexto + histórico, gere rascunho direto para [query]"
```

**PASSO 2 — REFINAMENTO:**
```
Modelo: Claude Haiku 4.5
Max tokens: 400
Prompt: "Refine este rascunho aplicando:
  - Não repita refutações: [refutedPoints]
  - Tom Worion: sem atendimento
  - Verifique afirmações vs toolResults
  - Sem linha dupla entre parágrafos"
```

### Arquivos

**js/writer.js:**

Funções criadas:
- `callHaiku(messages, options)` — chama Anthropic API via `callAnthropicWithRetry`
- `generateAndRefine(query, context)` — execução do fluxo rascunho → refinamento

**Logs obrigatórios:**
```javascript
[WRITER] gerando rascunho para rota: focused_research
[WRITER] rascunho: 287 tokens
[WRITER] refutações ativas: 2
[WRITER] refinando...
[WRITER] resposta final: 384 tokens
```

### Integração em chat.js

Linha ~2005, logo APÓS extração do reply do modelo:

```javascript
const shouldUseGenerateAndRefine = [
  'focused_research',
  'comparative_research',
  'private_context_synthesis',
  'direct_answer'
].includes(executionRoute);

if (shouldUseGenerateAndRefine && typeof generateAndRefine === 'function' && reply) {
  const writerContext = {
    route: executionRoute,
    sources: agentResult?.fetchedPages || [],
    history: apiMessages || messages,
    toolResults: agentResult?.toolResults || {}
  };

  const refined = await generateAndRefine(content, writerContext);
  if (refined) reply = refined;
}
```

**Comportamento:**
- Rotas elegíveis: `focused_research`, `comparative_research`, `private_context_synthesis`, `direct_answer`
- Se `generateAndRefine` falhar: usa resposta original (graceful degradation)
- Trivial routes continuam sem refinamento (latência < 500ms)

### PROBLEMA 1 - Writer Haiku Falhando

**Causa:** API Anthropic via `callAnthropicWithRetry` retorna formato OpenAI normalizado (`choices[0].message.content`), não formato nativo Anthropic (`content[0].text`).

**Correção (writer.js linha ~248):**
```javascript
// Extrair de múltiplos formatos
if (data?.choices?.[0]?.message?.content) {
  result = data.choices[0].message.content;  // OpenAI normalizado ✅
} else if (data?.content?.[0]?.text) {
  result = data.content[0].text;  // Anthropic nativo
} else if (typeof data?.content === 'string') {
  result = data.content;
}
```

**Status:** ✅ Corrigido - Writer agora gera rascunho e refinamento

| `js/memory-cards-runtime.js` | `buildMemoryCardChatContext()`, `activateMemoryCardForChat()` | Validar contra Supabase V2 quando disponível. Retornar vazio/null se inválido. |
| `js/contextGuardian.js` | `searchMemoryCards()` | Buscar `memory_contexts.status='active'` primeiro, depois filtrar cards por `context_id` válido. Excluir órfãos da injeção. |

### Regra de Governança

- **Contextos válidos:** `memory_contexts.status = 'active'`
- **Cards válidos:** `memory_cards_v2` com:
  - `context_id` preenchido
  - `context_id` referencia contexto `status='active'`
  - `status != 'archived'` e `status != 'draft'`
- **Bloqueios:**
  - Cards órfãos (sem `context_id`) não podem ser ativados
  - Cards/contextos `archived` ou `draft` não entram no runtime
  - Validação em 4 camadas: load, UI filter, chat start, prompt injection
